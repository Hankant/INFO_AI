# EWA-inspired 模型与参数解释

## 1. 为什么用 EWA-inspired 框架

EWA 在这里不是一个事后加上的统计工具，而是整套研究的行为语言。我们的理论主张本来就是：

`重复反馈 -> 架构吸引力更新 -> 路径依赖 -> 收敛 -> 稳态`

因此模型必须直接描述：

- 不同架构如何因为经验而变得更有吸引力或更失去吸引力。
- AI error 与 human error 是否具有不同的更新权重。
- 早期冲击为什么会留下长期后果。

## 2. 状态变量与选择变量

对参与者 `i`、架构 `k ∈ {1,2,3,4,5}`、轮次 `t`，定义：

- `A_{i,k,t}`：第 `t` 轮结束时，参与者对架构 `k` 的 attraction。
- `s_{i,t}`：第 `t` 轮实际选择的架构。
- `N_{i,t}`：累计经验权重。

## 3. 初始 attraction

为避免把先验偏见误当作经验学习，建议显式建模初始 attraction：

```text
A_{i,k,0}
= γ_{k0}
 γ_{k1} Prior_AI_Attitude_i
 γ_{k2} Initial_Architecture_Preference_i
 u_{i,k}
```

解释：

- `γ_{k1}` 反映事前 AI 态度如何影响高/低 AI 架构的起点吸引力。
- `γ_{k2}` 反映参与者在正式任务开始前就偏好的架构。
- `u_{i,k}` 是个体层未观测异质性。

这一步非常重要，因为它把“路径起点”与“路径更新”明确分开。

## 4. 经验权重更新

```text
N_{i,t} = ρ_i N_{i,t-1} + 1
```

其中：

- `ρ_i` 越高，旧经验被保留得越多。
- `ρ_i` 越低，参与者越重视近期信息。

## 5. attraction 更新方程

建议使用标准 EWA 形式并加入架构粘性控制：

```text
A_{i,k,t}
= [φ_i N_{i,t-1} A_{i,k,t-1}
   [δ_i + (1-δ_i) I(s_{i,t}=k)] π_{i,k,t}] / N_{i,t}
```

其中：

- `φ_i`：既有 attraction 的保留强度。
- `δ_i`：未被选择架构获得 counterfactual 更新的权重。
- `π_{i,k,t}`：架构 `k` 在第 `t` 轮的即时效用信号。

直观上：

- `φ_i` 高，说明一旦形成某种架构偏好，就更不容易被后续信息推翻。
- `δ_i` 低，说明参与者主要从自己实际走过的路径学习，更容易形成 lock-in。

## 6. 选择方程

```text
P(s_{i,t+1}=k)
= exp[λ_i (A_{i,k,t} - η_i I(k ≠ s_{i,t}))]
 / Σ_j exp[λ_i (A_{i,j,t} - η_i I(j ≠ s_{i,t}))]
```

参数解释：

- `λ_i`：choice sensitivity。越高表示越倾向选择当前 attraction 最高的架构。
- `η_i`：generic switching cost。越高表示越不愿意换架构。

把 `η_i` 单独放进选择方程，是为了把“通用粘性/惯性”与“由 AI early error 引发的路径依赖”分开。

## 7. 即时效用信号的建议写法

为了识别 AI 与 human 的非对称学习，建议把即时效用分解为来源和符号都可区分的形式：

```text
π_{i,k,t}
= θ_0
 + θ_AI^+ * AI_Correct_{i,k,t}
 + θ_AI^- * AI_Error_{i,k,t}
 + θ_H^+  * Human_Correct_{i,k,t}
 + θ_H^-  * Human_Error_{i,k,t}
 + θ_INT  * Interception_{i,k,t}
 - θ_T    * Time_Cost_{i,k,t}
 - θ_R    * Review_Cost_{i,k,t}
```

关键含义：

- `θ_AI^-`：AI 出错带来的负向更新权重。
- `θ_H^-`：human analyst 出错带来的负向更新权重。
- `θ_AI^+` 与 `θ_H^+`：正确信号的正向更新权重。
- `θ_INT`：错误被拦截时，架构的纠错能力得到奖励。

## 8. 模型中的核心可检验命题

### 8.1 错误来源非对称

若 AI error 被更强烈惩罚，应满足：

```text
θ_AI^- < θ_H^- < 0
且 |θ_AI^-| > |θ_H^-|
```

### 8.2 恢复不对称

若“犯一次错”对 AI 的伤害大于“做对很多次”带来的恢复，应满足：

```text
|θ_AI^-| > θ_AI^+
```

这意味着 AI 错误造成的架构下调，并不会被后续正确信号等比例修复。

### 8.3 路径依赖形成条件

当以下条件同时存在时，更容易出现长期锁定：

- `θ_AI^-` 的绝对值较大。
- `φ_i` 较高，旧 attraction 难以被冲掉。
- `δ_i` 较低，参与者主要从自己实际走过的路径学习。
- `η_i` 较高，切换成本让已经形成的低 AI 路径更不易逆转。

## 9. 为什么 early AI error 会在模型里留下长期后果

在这个模型里，early shock 不是简单的“更早发生了一次负反馈”，而是通过三层机制留下痕迹：

### 9.1 早期 attraction 尚未稳定

早期阶段各架构 attraction 差距还小，一次大幅负更新更容易改写排序。

### 9.2 后续选择会改变后续暴露

如果 early AI error 使参与者转向低 AI 架构，那么后续接触高 AI 架构的机会减少，正向修复信号也会变少。

### 9.3 粘性会把已形成路径保留下来

即使后续 AI 的客观表现恢复，高 `φ_i` 和 `η_i` 也会让参与者继续待在已经形成的低授权路径上。

这三步合起来，才构成真正的 path dependence。

## 10. counterfactual payoffs 的使用建议

因为实验平台预先知道每个案例中 AI 与 human analyst 的潜在正确性，所以研究者可以计算每轮五种架构的潜在 payoffs。

这允许做两种估计：

### 10.1 保守估计

- 令 `δ_i` 接近 0。
- 只强调参与者对已选择架构的经验学习。
- 更贴近“只从自己走过的路径学”的解释。

### 10.2 扩展估计

- 允许 `δ_i` 由数据估计。
- 使用研究者计算的 counterfactual payoffs 对未选择架构进行虚拟更新。
- 适合检验参与者是否在做某种隐含的 belief learning。

正式论文可以把 `δ_i = 0` 的模型作为主规格，把允许 `δ_i > 0` 的模型作为鲁棒性扩展。

## 11. 估计建议

建议采用层级贝叶斯估计或层级最大似然估计：

- 个体层允许 `λ_i`、`φ_i`、`ρ_i`、`η_i` 和关键 `θ` 参数异质。
- 条件层比较不同处理组的参数后验分布或均值差异。

主报告重点不应只是“模型拟合得好不好”，而应是参数与理论命题是否一一对应：

- `θ_AI^-` 是否明显更负。
- `AI Early Error` 条件下是否更容易估计出高持久性、低恢复速度。
- 控制 `η_i` 后，路径差异是否仍然存在。

## 12. 与研究问题的对应关系

| 研究问题 | 对应参数或量 |
|---|---|
| AI error 是否被更强惩罚 | `θ_AI^-` 与 `θ_H^-` |
| 后续正确是否足以修复早期伤害 | `θ_AI^+` 与 `|θ_AI^-|` |
| 已形成路径是否容易被改写 | `φ_i`、`ρ_i` |
| 是否存在一般性粘性 | `η_i` |
| 参与者是否只从自己走过的路径学习 | `δ_i` |

因此，EWA-inspired 模型在本文中的功能不是“补充性估计”，而是把长期架构学习的每一个理论部件都参数化。
