# Agent A 阶段门与抽查清单

## 一、监管方式

Agent A 采用阶段门监管，而不是等所有材料都写完再一次性审查。每个阶段都必须通过抽查，未通过就退回修改。

---

## 二、阶段门设置

### 阶段门 1：选题边界关

**必须回答**

1. 研究对象是不是“人机协作架构学习”？
2. 是否明确区分 trust、reliance、calibration、architecture learning？
3. 是否明确提出 path dependence 和 convergence？

**抽查不通过的典型表现**

1. 题目还是 adoption 语气。
2. 研究问题还是 AI 出错后信任下降。
3. 没有长期稳定状态概念。

### 阶段门 2：理论机制关

**必须回答**

1. 机制链条是否闭环？
2. EWA 是否贯穿全文？
3. early AI error 为什么能造成长期后果？

**抽查不通过的典型表现**

1. 只有概念，没有机制。
2. 只有 EWA 名词，没有对应变量。
3. 只有“AI error 更严重”的口号，没有路径依赖逻辑。

### 阶段门 3：识别设计关

**必须回答**

1. AI 与 human 的长期表现是否严格可比？
2. 错误时点与错误来源是否被清楚操控？
3. 稳定状态是否可测量？
4. 客观最优架构是否可定义？

**抽查不通过的典型表现**

1. 条件设计不能区分 early 和 late effect。
2. 客观最优架构没有效用函数支撑。
3. 因变量还是 trust score。

### 阶段门 4：会议表达关

**必须回答**

1. 一页摘要能否讲清研究转向？
2. 图示是否让人一眼看到核心机制？
3. 汇报时是否会被误认成 algorithm aversion old wine in new bottle？

**抽查不通过的典型表现**

1. PPT 中 trust 出现次数远高于 architecture。
2. 图只讲“AI 出错 -> 信任下降”。
3. 结论没有治理含义。

## 三、Agent A 抽查问题库

### 抽查 Agent B 时优先问

1. 你的实验到底是在测“角色学习”还是“建议采纳”？
2. 为什么 80 轮足够观察 convergence？
3. 为什么 early AI error 和 late AI error 在理论上应该不同？
4. 客观最优架构是谁定义的？是否依赖研究者主观设定？
5. 如果参与者一开始就讨厌 AI，怎么区分先验偏见和路径依赖？

### 抽查 Agent C 时优先问

1. 你的整合版是不是削弱了理论锋芒？
2. 你的图示是否真正体现了 attraction updating？
3. 你的摘要有没有把研究写成 trust 研究？
4. 你的 PPT 如果给 CHI/FAccT/CSCW 听众讲，最容易被质疑哪一点？

## 四、阶段性通过标准

### 通过标准 A：理论表达

如果 3 分钟内不能清楚说出：

```text
从 trust/adoption 转向 architecture learning
从短期错误反应转向长期收敛路径
从学习 AI 能力转向学习 AI 角色
```

则阶段不通过。

### 通过标准 B：方法可识别

如果外部听众不能理解为什么这个设计能识别 path dependence，而不是普通的 error response，则阶段不通过。

### 通过标准 C：学术锋芒

如果最终稿只能作为“导师内部讨论材料”，而不能作为会议前汇报材料，则阶段不通过。
